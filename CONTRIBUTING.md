# Contributing Guidelines

## Branch Strategy

### Main Branch
- The `main` branch contains approved research materials only.
- Direct commits to `main` are restricted.

### Draft Branches
Use:

`draft/<research-topic>`

Examples:
- draft/literature-review
- draft/competitive-analysis
- draft/client-interview

### Fix Branches
Use:

`fix/<issue-number>`

Examples:
- fix/08
- fix/19



## Contribution Workflow

1. Update local repository
2. Create a draft branch
3. Add or modify research files
4. Commit changes properly
5. Push branch to GitHub
6. Create Pull Request
7. Obtain approval
8. Merge into `main`



## Commit Message Convention

Format:
`<type>: <description>`

Examples:
- research: added competitor analysis
- docs: updated interview transcript
- fix: corrected reference formatting



## Pull Request Rules

- PR review is mandatory
- Minimum 1 approval required
- Ensure files are properly organized
- Avoid duplicate research materials



## Repository Rules

- Use proper naming conventions
- Keep references organized
- Store only relevant research materials
- Do not upload temporary files
